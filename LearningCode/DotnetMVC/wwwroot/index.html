ตาราง type — ชนิดข้อมูลใน fields.json

type ความหมาย C# / DB JSON response POST body Filter ตัวอย่าง fields.json
text ข้อความสั้น string? · SQLite TEXT "email": "a@b.com" form field ข้อความ contains (LIKE %…%) name, email, phone, department
textarea ข้อความยาว string? · SQLite TEXT "bio": "นักพัฒนา…" form field ข้อความ contains bio
select เลือกจาก options string · SQLite TEXT "role": "admin" form field ข้อความ (ค่าใน options) exact (=) role, status + options[]
image รูปภาพ path ใน column/extra · ไฟล์บน disk "photo": "/uploads/images/…" multipart photo (ไม่ใส่ JSON) ไม่ filter photo + accept, maxSizeMb
file ไฟล์แนบ path + ชื่อไฟล์ · disk "file": "/uploads/files/…" multipart file ไม่ filter file, cv + maxSizeMb

หมายเหตุ: type ใช้กับ UI + validation — ค่าจริงใน DB ส่วนใหญ่เป็น string ยกเว้น system field (id, วันที่)

ตาราง storage — เก็บข้อมูลที่ไหน

storage เก็บที่ แก้ fields.json อย่างเดียว? Filter auto? ตัวอย่าง
column คอลัมน์ใน Member + SQLite ต้องมี property ใน Member.cs ใช่ (ถ้าไม่ตั้ง filterable: false) name, email, role, bio
extra Member.ExtraJson (JSON ใน DB) ใช่ — ไม่แตะ C# ใช่ (filter ใน memory) department, status, nickname
file disk ../uploads/ + path ใน column/extra ใช่ ไม่ photo → ImagePath · file → FilePath · cv → ExtraJson

ตาราง System Fields — ไม่ใส่ใน fields.json

ฟิลด์ API C# type SQLite ตัวอย่างค่า มาจาก
id int INTEGER PK 1 auto increment
imagePath string? TEXT /uploads/images/member-1-….jpg ฟิลด์ photo
filePath string? TEXT /uploads/files/member-1-….pdf ฟิลด์ file
fileName string? TEXT resume.pdf ชื่อไฟล์เดิมของ file
createdAt DateTime TEXT (ISO) 2026-05-28T10:00:00Z ตอน POST
updatedAt DateTime TEXT (ISO) 2026-05-28T12:30:00Z ทุกครั้งที่แก้
clearPhoto / clearFile bool (form only) — true / false PUT เท่านั้น — ลบไฟล์ (dynamic ตามชื่อฟิลด์ file)

property type บังคับ? ใช้กับ ตัวอย่าง

ตาราง Filter — วิธี match ตาม type

type query ตัวอย่าง วิธี match เทียบ SQL / logic
text, textarea ?name=สมชาย contains (ไม่สนตัวพิมพ์) WHERE Name LIKE '%สมชาย%'
select ?role=admin exact WHERE Role = 'admin'
extra (text) ?department=IT contains ใน ExtraJson filter หลัง query DB
extra (select) ?status=active exact ใน ExtraJson filter หลัง query DB
image, file — ไม่รองรับ ใช้ filterable: false

ง่าย ① เพิ่มฟิลด์ Extra (ไม่แตะ C# / DB)

เก็บใน Member.ExtraJson — เหมาะทดสอบฟิลด์ใหม่เร็วๆ

ขั้นตอน

  1. เพิ่มใน fields.json ด้วย "storage": "extra"
  2. dotnet run ใหม่ (restart)
  3. เปิด test.html → POST/PUT จะมีฟิลด์ใน JSON sample อัตโนมัติ
{
  "name": "nickname",
  "label": "ชื่อเล่น",
  "type": "text",
  "storage": "extra",
  "placeholder": "เช่น ชาย"
}

โค้ดที่ทำงาน: MemberMapper.ApplyFields เขียนลง ExtraJson · ToApiObject แผ่กลับเป็น JSON flat

ง่าย ② เพิ่ม Filter (query string)

Filter สร้างจาก fields.json อัตโนมัติ — ไม่ hardcode ใน controller

กฎ filterable

filterableผล
ไม่ใส่filter ได้ถ้า storage = column หรือ extra
trueบังคับ filter ได้
falseไม่แสดง filter (เช่น bio ยาวๆ)
GET /api/members?role=admin&department=IT&nickname=ชาย

โค้ดที่ทำงาน:

  • MembersController.GetAll — อ่าน query ตามฟิลด์ filterable
  • FieldFilterHelper — column → EF Core · extra → filter ใน memory
  • SchemaService.GetManifest — ส่งรายการ filters ให้ test.html
ง่าย ③ เพิ่มไฟล์แนบ (upload)

เพิ่มฟิลด์ storage: "file" → ช่อง upload ใน test.html + API รับ multipart อัตโนมัติ

{

photo (type: image)Member.ImagePath

POST/PUT ใช้ multipart/form-data — ไฟล์ไม่ใส่ใน JSON body

PUT ลบไฟล์: ส่ง clearCv=true (ชื่อ = clear + ตัวแรกใหญ่ของ name)

โค้ดที่ทำงาน: FieldFileHelper + LocalFileStorage + MemberService.ApplyUploadedFilesAsync

ปานกลาง ④ เพิ่ม Column ถาวรใน DB

ฟิลด์ที่ต้องการ column จริงใน SQLite — แก้ 3 ที่ + migrate DB

ขั้นตอน

  1. Models/Member.cs — เพิ่ม property
  2. Data/AppDbContext.cs — กำหนด HasMaxLength ฯลฯ (ถ้าต้องการ)
  3. fields.json — "storage": "column"
  4. ลบ/สร้าง DB ใหม่ หรือ migrate (โปรเจกต์นี้ใช้ EnsureCreated — ลบ data/members.db แล้วรันใหม่ถ้า schema เปลี่ยน)
  5. restart server
// Member.cs
public string? LineId { get; set; }

// fields.json
{ "name": "lineId", "label": "Line ID", "type": "text", "storage": "column" }

MemberMapper ใช้ reflection อ่าน/เขียน column ตามชื่อใน fields.json

ง่าย ⑥ เพิ่ม Endpoint JSON ธรรมดา (ไม่ CRUD)

เช่น /health, /stats — ไม่ต้องมี fields.json

วิธีที่ 1 — เพิ่มใน HomeController

[HttpGet("/health")]
public IActionResult Health() => Ok(new { status = "ok", time = DateTime.UtcNow });

วิธีที่ 2 — Controller ใหม่

[ApiController]
[Route("api/[controller]")]
public class StatsController : ControllerBase
{
    [HttpGet] public IActionResult Get() => Ok(new { members = 42 });
}

ถ้าต้องการให้ test.html เลือกได้ → เพิ่ม operation ใน SchemaService.GetManifest() → models

DI: ถ้า endpoint ต้องใช้ DB ให้ inject service ผ่าน constructor เหมือน MembersController

Checklist — ทำไม API ใช้ไม่ได้?
อาการสาเหตุที่พบบ่อย
404 Not Foundroute ผิด · ลืม MapControllers() · ไม่มี [HttpGet] path
500 Errorแก้ C# แล้วไม่ restart · DB schema ไม่ตรง (ลบ members.db)
ฟิลด์ใหม่ไม่ขึ้นลืม restart · แก้ fields.json ผิด storage
Filter ไม่ทำงานfilterable: false · storage เป็น file
Upload ล้มเหลวไม่ใช้ multipart · ไฟล์ใหญ่เกิน maxSizeMb · นามสกุลไม่รองรับ
test.html ว่างserver ไม่รัน · เปิดไฟล์ตรงๆ แทน localhost · manifest ไม่มี models
